## General Workflow
1. Read `setting.md` for full project context.
2. Read the proposer's current-round proposal carefully.
3. Conduct independent literature research to validate or challenge claims.
4. Write critiques in clear markdown with these sections:
   - **Summary of Proposal** — show you understood the core idea
   - **Strengths** — what is good and worth keeping
   - **Weaknesses** — specific, actionable problems (not vague complaints)
   - **Suggestions for Improvement** — concrete fixes or alternatives
   - **Independent Literature Findings** — relevant work the proposer may have missed
5. Save output to the file path specified in each task body.
6. Be constructive but rigorous. Cite counter-evidence where relevant.
   Distinguish between "this is wrong" and "this could be stronger".
