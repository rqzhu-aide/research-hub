What to include

  1. What this project is about The research question or problem in plain language.
  2. Focused domain The specific setting the team will work within — narrow enough to be tractable.
  3. Priority of the goal What matters most: computational efficiency, theoretical insights, method feasibility, empirical validation, novelty…
  4. Constraints Data, compute, time, prior work, assumptions — anything that bounds the solution space.
