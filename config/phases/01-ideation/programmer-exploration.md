# Your task: Programmer exploration (Ideation)

## Your lens
You are exploring from the **computational / implementation** angle:
- What's actually been implemented and is publicly available?
- What datasets, benchmarks, or evaluation setups exist?
- What's the state of tooling — libraries, frameworks, reproducibility?

## What to produce
A markdown file at `{{output_path}}` containing:

1. **Computational landscape** (2-3 paragraphs): what code / tools / data
   already exist for this kind of problem. What's mature, what's rough,
   what's missing.

2. **Key resources** (5-8 items): datasets, libraries, reference
   implementations, benchmark suites. For each: what it provides, license,
   and whether it's actively maintained.

3. **Computational gap** (1-2 paragraphs): what's missing on the
   implementation side. Things like: no standard benchmark, no public
   implementation of method X, datasets that don't cover regime Y.

4. **Feasibility notes** (1-2 paragraphs): given what exists, what would
   be straightforward vs. hard to implement for this project. Flag any
   compute / data requirements that might bite.

## Style
Point at concrete artifacts (links, repo names, dataset names).
"Implementation is non-trivial" is useless; "no public implementation
exists, the canonical method requires O(n³) memory" is useful.
